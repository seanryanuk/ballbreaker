# Ballbreaker - Agentic Context & Guidelines

Welcome to the Ballbreaker codebase context. This file ensures that subsequent AI agent sessions can align with the project design, conventions, and architectural rules immediately.

## 1. Project Overview
**Ballbreaker** is a Linux GUI application (with a fallback CLI) designed to manage target installations of software distributed as tarballs (`.tar.gz`, `.tar.xz`, `.tar.bz2`, etc.).

- **Extraction Target**: Configurable (defaults to `/opt` or a user-specified custom path like `~/Apps`).
- **Launcher Integration**: Creates standard `.desktop` files in a configurable directory (defaults to `~/.local/share/applications` or custom path) and links the binaries to `~/.local/bin` or `/usr/local/bin`.
- **Administrative Privileges**: Runs in user space. Privileged commands (writing to `/opt` or `/usr/local/bin`) must execute via Polkit (`pkexec`) or standard `sudo` command wrappers, rather than running the GUI root-level.

## 2. Tech Stack & Environment
- **Runtime**: Python `>= 3.14`
- **Package Manager**: `uv` (Astral)
- **GUI Framework**: PySide6 (Qt 6 for Python)
- **Styling**: Modern, premium dark-themed layout using custom QSS (Qt Style Sheets) and high-quality widgets.
- **Testing**: `pytest`

## 3. Directory Layout
All source code resides inside the `src/ballbreaker/` directory structure:
```
src/
└── ballbreaker/
    ├── __init__.py
    ├── __main__.py          # App runner (arg parse & GUI/CLI selection)
    ├── cli.py               # Headless CLI implementation
    ├── core/                # Core Business Logic (NO GUI dependencies here)
    │   ├── __init__.py
    │   ├── extractor.py     # Safe extraction, symlinks, privilege check/elevation
    │   ├── desktop_entry.py # Generating and writing standard .desktop entries
    │   └── config.py        # Persistent user settings and directory path overrides
    └── gui/                 # Presentation / View Layer (PySide6)
        ├── __init__.py
        ├── app.py           # Main Window Layout
        ├── stylesheet.py    # Dark theme QSS definitions
        └── widgets/         # Custom UI elements (e.g. dropzones, path selectors)
```

## 4. Architectural Rules & Best Practices
- **Strict Separation of Concerns**: Keep Qt GUI code completely decoupled from the logic in `core/`. Core modules must be testable from raw CLI / scripts without initializing a QApplication.
- **Elevation of Privilege**: Implement elevation wrappers in `extractor.py`. When copying files or creating directories in `/opt` or `/usr/local/bin`, check if write access is available. If not, use `pkexec` (preferred on GUI desktops) or fallback to `sudo` via standard library `subprocess`.
- **Desktop Entry Integrity**: Ensure generated `.desktop` files follow the [XDG Desktop Entry Specification](https://specifications.freedesktop.org/desktop-entry-spec/latest/).
- **Dark Mode Aesthetics**: The GUI should follow a high-end dark palette. Use deep slate grays (`#1e1e2e`), vibrant accent colors (e.g., violet `#8839ef` or cyan `#04a5e5`), and generous padding (10-15px). Avoid browser/desktop default system buttons if they look outdated; style them cleanly using QSS.
